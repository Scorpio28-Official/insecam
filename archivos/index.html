Primeros pasos con Ajax

Log in or subscribe for free to enjoy all this course has to offer!

En este capítulo aprenderás a utilizar Ajax a través de jQuery, que simplifica la forma en la que obtener datos del servidor para crear páginas dinámicas.

En la tercera parte del curso Crea páginas web interactivas con JS, se trata el uso de Ajax utilizando JavaScript (sin jQuery). En dicho curso, se explican conceptos fundamentales para el uso de Ajax, como el protocolo HTTP o el formato de texto JSON para la transmisión de información. Si quieres profundizar en dichos conceptos, echa un vistazo a ese curso.

¿Qué es Ajax?

Ajax es una técnica de desarrollo que permite actualizar ciertas partes de una página web, mejorando la interactividad y la experiencia de usuario. Cuando estás navegando en una web tradicional y cambias de página, se producen las siguientes acciones:

  1. Se envía una petición al servidor para obtener la nueva página.

  2. El servidor gestiona la petición y envía la página (archivos HTML, CSS, imágenes, etc.).

  3. Se muestra la información en la navegador.

Esta ha sido la forma de funcionar desde el principio de la Web, pero con la aparición de Ajax, es posible recargar sólo una parte de la página y no interrumpir constantemente la navegación:

  1.  En primer lugar, se envía una petición al servidor con el objetivo de obtener los datos necesarios.

  2. El servidor procesa los datos demandados y envía una respuesta. 

  3. Se reciben los datos y se muestran en una parte de la página, sin recargarla en su totalidad.

Aunque el funcionamiento tradicional sigue siendo válido en la mayoría de los casos, se dan muchas situaciones en las que el uso de Ajax es muy útil. Por ejemplo, se puede utilizar Ajax para validar datos de un formulario en tiempo real, para mostrar sugerencias como hace google en su buscador, o para cargar imágenes en una galería cuando el usuario se va desplazando hacia abajo.

El término Ajax es un acrónimo de "Asyncronous JavaScript And XML" (JavaScript asíncrono y XML), ya que cuando Ajax se inventó, el formato estándar para la transferencia de información era XML. Más adelante, se adoptó el formato JSON, que toma la sintaxis de los objetos de JavaScript, y es más corto y fácil de leer. Por lo tanto, Ajax se utiliza para pedir datos al servidor con el lenguaje JavaScript. Estos datos se retornan de forma asíncrona en formato JSON.

Realizando peticiones Ajax

Ahora que ya sabes en qué consiste Ajax, necesitamos un servidor al que enviarle peticiones y obtener información. Para ello, utilizaremos un servidor, ya preparado, con una API que retorna información JSON. De esta forma, no tenemos que perder tiempo creando un servidor que responda a nuestras peticiones para poder utilizar Ajax.

Peticiones GET

Reqres es un servidor con una API al que puedes enviar peticiones, y que retorna información en formato JSON para realizar simulaciones con Ajax.

Por ejemplo, si en el navegador vamos a https://reqres.in/api/users, obtenemos una lista de usuario ficticios en formato JSON. La información que obtenemos de los usuarios es su nombre, apellido y foto:

Veamos cómo utilizar jQuery para enviar peticiones Ajax y obtener esta información :)

La función de jQuery   $.get()  permite obtener datos del servidor con una petición HTTP con el método GET. La sintaxis es la siguiente: 

$.get(dirección, function(respuesta) {
  // Acciones a realizar cuando se obtiene una respuesta
});

Los dos parámetros que se pasan a la función $.get son los siguientes:

  • La dirección es la dirección url de servidor al que vamos a pedir la información.

  • La función callback se ejecuta cuando la petición se haya realizado con éxito.

  • El parámetro respuesta contiene la información retornada por el servidor. Recibimos esta información como parámetro de la función callback.

Peticiones POST

La función  $.post()  sirve para enviar datos al servidor. Por ejemplo, se puede utilizar esta función para crear una nuevo usuario a través de un formulario. Los datos del formulario se envían al servidor y se guardan en la base de datos. La sintaxis es la siguiente:

$.post(dirección, {objeto_con_datos}, function(datos) {
  // Acciones a realizar cuando se obtiene una respuesta
});

  • La dirección es la dirección url de servidor al que vamos a enviar la información.

  • La función callback se ejecuta cuando la petición se haya realizado con éxito.

  • El parámetro datos contiene la información registrada en el servidor Recibimos esta información como parámetro de la función callback.
